h1. ISO8583 AMEX Decoder – Usage Guide

h2. Overview

This module provides a *schema-driven ISO8583 decoder* for AMEX messages.

It supports:
* MTI-based message resolution
* Binary bitmap parsing
* Composite field decoding
* Dynamic schema adjustments via *DecodeOptions*
* Mapping to *typed message objects* (e.g. Amex1100Message)
* Pretty-print debugging output

---

h2. Key Components

h3. 1. AmexDecoder

Main entry point for decoding messages.

Responsibilities:
* Reads MTI and bitmaps
* Resolves message definition
* Applies field definition adjusters (optional)
* Decodes fields
* Maps result into typed message object

---

h3. 2. DecodedAmexResult<T>

Result of decoding.

Contains:
* *message* → typed POJO (e.g. Amex1100Message)
* *flatValues* → all decoded values (for debug/logging)
* *fieldMeta* → offsets, raw hex, lengths
* *effectiveFieldDefinitions* → schema after adjusters applied

---

h3. 3. Message Definitions

Examples:
* Amex1100Definition
* Amex1110Definition

Define:
* Field structure
* Encoding
* Length rules
* Subfields (composite fields)

---

h3. 4. Adjusters (FieldDefinitionAdjuster)

Allow dynamic schema changes based on flags.

Use cases:
* Different EMV variants
* Conditional subfields
* Changing length/encoding rules

---

h3. 5. DecodeOptions

Optional flags that influence decoding.

Example:
{code:java}
DecodeOptions options = DecodeOptions.empty()
.with(DecodeOptionKeys.DE55_VARIANT, "SIMPLIFIED");
{code}

---

h3. 6. DecodedPrettyPrinter

Utility for formatted debug output.

Supports:
* ANSI highlighting (can be turned off)
* Bitmap visualization
* Subfield structure
* Raw + meta output
* Highlighting fields by PID or field name

---

h2. Basic Usage

h3. Decode Message

{code:java}
AmexDecoder decoder = new AmexDecoder();

DecodedAmexResult<? extends AmexMessage> result =
decoder.decode(rawMessage);
{code}

---

h3. Decode with Options

{code:java}
DecodeOptions options = DecodeOptions.empty()
.with(DecodeOptionKeys.DE55_VARIANT, "LEGACY");

DecodedAmexResult<? extends AmexMessage> result =
decoder.decode(rawMessage, options);
{code}

---

h3. Access Typed Message

If MTI is known:

{code:java}
DecodedAmexResult<Amex1100Message> result =
decoder.decode(rawMessage, new Amex1100Definition(), DecodeOptions.empty());

Amex1100Message message = result.getMessage();

String pan = message.getPan();
String stan = message.getStan();
{code}

If MTI is dynamic:

{code:java}
AmexMessage message = result.getMessage();

if (message instanceof Amex1100Message) {
Amex1100Message m = (Amex1100Message) message;
}
{code}

---

h2. Debug Output

{code:java}
DecodedPrettyPrinter.print(
result,
rawMessage,
System.out,
Arrays.asList("P2", "PAN", "POS_DATA_CODE.CARD_PRESENT")
);
{code}

---

h3. Highlighting

You can highlight fields using:

* *PID format* → P2, P55.1
* *Field names* → PAN, POS_DATA_CODE.CARD_PRESENT

Both formats are supported.

---

h2. Disable ANSI Colors

For plain text output (e.g. logs, CI):

{code:java}
DecodedPrettyPrinter.setAnsiMode(DecodedPrettyPrinter.AnsiMode.OFF);
{code}

---

h2. Runner Example

{code:java}
public class DecodeRunner {

    public static void main(String[] args) {

        byte[] rawMessage = ... // from DB / blob

        AmexDecoder decoder = new AmexDecoder();

        DecodeOptions options = DecodeOptions.empty()
                .with(DecodeOptionKeys.DE55_VARIANT, "SIMPLIFIED");

        DecodedAmexResult<? extends AmexMessage> result =
                decoder.decode(rawMessage, options);

        DecodedPrettyPrinter.setAnsiMode(DecodedPrettyPrinter.AnsiMode.ON);

        DecodedPrettyPrinter.print(
                result,
                rawMessage,
                System.out,
                Arrays.asList("PAN", "P22.6", "PAYMENT_ACCOUNT_DATA")
        );
    }
}
{code}

---

h2. Important Notes

h3. 1. Schema Adjustments
* Applied only during decoding
* Stored in effectiveFieldDefinitions
* Printer uses this snapshot (no need to pass options again)

h3. 2. Optional Flags Behavior

If no flags are provided:
* Base schema is used
* No behavior changes

---

h2. Summary

The decoder is:
* Schema-driven
* Extensible via flags
* Safe by default (no flags → no changes)
* Debug-friendly

Additional features:
* Flexible field highlighting (by name or PID)
* ANSI output can be enabled or disabled
